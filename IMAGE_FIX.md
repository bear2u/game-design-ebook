# 이미지 경로 문제 해결 가이드

## 현재 상황

- ✅ 이미지 경로 처리 로직: 수정 완료
- ⚠️ `imgs/` 폴더: **없음** (생성 필요)
- ⚠️ 이미지 파일: **없음** (추가 필요)

## 이미지 파일 위치

원본 `book.md` 파일에서 이미지 경로는 다음과 같습니다:
```
imgs/img_in_image_box_*.jpg
```

번역된 챕터 파일에서는:
```
../imgs/img_in_image_box_*.jpg
```

## 해결 방법

### 1. 이미지 폴더 및 파일 추가

프로젝트 루트에 `imgs` 폴더를 생성하고 이미지 파일들을 추가하세요:

```bash
# imgs 폴더 생성
mkdir imgs

# 원본 PDF나 소스에서 이미지 파일들을 imgs/ 폴더에 복사
# 파일명 형식: img_in_image_box_*.jpg
```

### 2. 이미지 경로 확인

브라우저 콘솔(F12)을 열고 챕터를 로드하면 다음 정보가 표시됩니다:
- 현재 페이지 경로
- 계산된 이미지 경로
- 이미지 로드 성공/실패 메시지

### 3. 이미지가 없는 경우

이미지 파일이 없어도 텍스트는 정상적으로 표시됩니다. 이미지 부분만 표시되지 않습니다.

## 디버깅

브라우저 콘솔에서 확인할 수 있는 정보:
- `이미지 경로 디버깅`: 경로 계산 정보
- `이미지 로드 실패`: 실패한 이미지 경로
- `이미지 로드 성공`: 성공한 이미지 경로

## 예상 이미지 경로

- 로컬: `http://localhost:8000/imgs/img_in_image_box_*.jpg`
- GitHub Pages: `https://[username].github.io/game-design-translate/imgs/img_in_image_box_*.jpg`
